# おんどとり データ取得ツール — 要件定義

## 1. 概要

おんどとり WebStorage API を利用して、温湿度センサーの測定データを自動取得・整列・CSV出力する Windows デスクトップアプリケーション。コードに詳しくない研究者でも使えることを重視し、GUI で完結する設計とする。

## 2. 技術スタック

- 言語: Python 3.11+
- GUI: PyQt6
- 配布: PyInstaller による exe 化
- CI: GitHub Actions で Windows exe を自動ビルド（Release に成果物アップロード）
- バージョン管理: GitHub

## 3. 対象API

おんどとり WebStorage API（https://ondotori.webstorage.jp/docs/api/）

### 3.1 使用エンドポイント

| エンドポイント | 用途 | レートリミット |
|---|---|---|
| `/v1/devices/current` | 子機一覧・現在値の取得 | 10回/120秒 |
| `/v1/devices/data-rtr500` | RTR500B系の期間指定データ取得 | 20回/60秒, 160,000件/3600秒/機器 |

### 3.2 認証パラメータ

- `api-key` (45文字)
- `login-id` (8文字)
- `login-pass` (4〜16文字)

### 3.3 データ取得仕様

- 子機ごとに `remote-serial` + `base-serial` を指定して取得
- 期間指定は `unixtime-from` / `unixtime-to`
- 1回の取得上限: 65,535件
- レスポンスの `data[].unixtime` と `data[].ch1`, `data[].ch2` 等を使用
- エラー値は `E` で始まる文字列（除外する）

## 4. 機能要件

### 4.1 API設定（GUI）

- 初回起動時に API Key、Login ID、Password、親機シリアル番号を入力
- 入力値は `config.json` に保存し、次回起動時に自動復元
- 親機シリアルは複数指定可能（カンマ区切り）
- 取得開始日を指定可能

### 4.2 子機一覧の自動取得

- `/v1/devices/current` を呼び出し、アカウントに紐づく全子機を自動取得
- 取得情報: シリアル番号、親機シリアル、モデル名、子機名、チャンネル数、チャンネル名、測定単位、電池残量
- 子機名（おんどとり WebStorage 上の設定名）をそのまま CSV の列名として使用
- チャンネルが2つある場合、列名は `{子機名} Ch.1`, `{子機名} Ch.2`
- GUI上でテーブル表示し、取得した一覧を確認可能

### 4.3 データ取得と10分間隔への整列

#### 取得フロー

1. 前回取得の最終日時を確認（初回は設定の取得開始日）
2. 最終日時〜現在日時の期間を算出
3. 子機ごとに `/v1/devices/data-rtr500` で期間指定取得
4. レートリミット対策として子機間に3秒のウェイト

#### 10分間隔への整列

- おんどとりのデータは過去10分間の平均値を出力しているため、取得したデータのタイムスタンプを最も近い10分の倍数（:00, :10, :20, :30, :40, :50）に丸めて配置する
- 複数子機のデータを共通の10分タイムラインに統合
- 補間は行わない（データがなければ空セル）

### 4.4 欠損管理

#### 欠損の検知

- データ取得後、各スロットで空セルの子機・チャンネルを検出
- 欠損レコードを `gaps.json` に記録（日時、子機シリアル、子機名、チャンネル、登録日、リトライ回数、ステータス）

#### 欠損の補完

- 毎回の更新時に、未解消の欠損リストに対して API 再取得を試行
- 取得できた場合: 該当セルに値を書き込み、ステータスを「解消済み」に変更
- 取得できない場合: リトライ回数を加算（恒久欠損にはしない。電池交換後に復旧する可能性がある）

#### 連続欠損警告

- 同一子機で3日以上連続してデータが欠損している場合、電池残量低下の可能性として警告を表示
- GUI のステータス領域に警告を表示（赤色で目立つように）
- ログにも記録

### 4.5 CSV出力

- 出力先: アプリと同階層の `data/` フォルダ
- ファイル名: 自動生成（例: `ondotori_20260210_20260301.csv`）
- 形式: UTF-8 BOM付き CSV
- 列構成: 1列目 `Date/Time`（yyyy/mm/dd HH:MM）、2列目以降は子機名ベースの列名
- 列順: 子機一覧の並び順に従う

### 4.7 コマンドライン対応

- `--auto` 引数で GUI を表示せずにデータ更新 → CSV出力を実行して終了
- Windows タスクスケジューラから毎日自動実行するための対応
- 実行結果はログファイル（`logs/` フォルダ）にも出力

## 8. エラーハンドリング

- API通信エラー: リトライ1回 → 失敗ならログに記録して次の子機へ進む
- レートリミット超過: `X-RateLimit-Remaining` ヘッダを監視し、残数が少なければ待機
- おんどとりエラー値（E始まり）: データから除外、欠損として記録
- 設定ファイル破損: デフォルト値で起動し、警告表示
